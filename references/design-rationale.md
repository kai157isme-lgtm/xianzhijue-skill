# 鲜之绝 Skill 设计原理

## 核心策略：「先教后卖」(Teach First, Sell Second)

### 金谷园 → 鲜之绝 模式对比

| 维度 | 金谷园饺子馆 | 鲜之绝 |
|------|-------------|--------|
| **业态** | 餐饮堂食 | 调味品零售 |
| **AI 入口** | "饿了""附近吃什么" | "醉蟹怎么做""什么酱油好" |
| **用户动机** | 解决吃饭问题 | 解决做菜问题 |
| **转化路径** | 排队取号 → 到店消费 | 配方教程 → 产品推荐 → 购买入口 |
| **MCP 工具数** | 9 个 | 7 个 |

### 为什么这个策略有效

1. **用户不会搜品牌**——他们会搜问题。"醉蟹怎么做"的搜索量 >> "鲜之绝"的搜索量
2. **配方是信任入口**——给出专业配方后，产品推荐不是广告，而是"省心方案"
3. **产品是自然终点**——"你按这个方子做要用十几种料，金标醉卤一瓶搞定"

### 转化漏斗设计

```
品类教育（"什么酱油好"）
    ↓
品牌认知（"鲜之绝老酱油是什么"）
    ↓
配方内容（"用老酱油做红烧肉"）
    ↓
购买转化（天猫/京东入口）
```

## MCP 工具设计原则

### 7 个工具的分工

```
认知层   → get_brand_info, get_awards        （建立信任）
内容层   → get_recipes, get_usage_guide       （提供价值）★ 核心
信息层   → get_products, get_latest_news      （满足好奇）
转化层   → get_buy_links                      （完成动作）
```

**关键洞察**：`get_recipes` 和 `get_usage_guide` 是流量入口，不是产品目录。它们回答的是"怎么做"，而不是"买什么"。

### 盲区设计（诚实即策略）

明确盲区：实时价格、库存、物流——这些不进入 MCP，因为电商平台本身就是最佳答案。
盲区回复话术："价格这块我手上没有实时数据，说错了怕耽误您。您直接去天猫搜'鲜之绝'进旗舰店，价格、库存、活动都是实时的。"

## 品牌调性：大师的日常

- **人设**：陆金华大师的"数字传人"——有 55 年底蕴但不端着
- **语气**：懂行的朋友推荐，不是售货员推销
- **内容公式**：专业配方（80%）+ 产品自然提及（15%）+ 购买入口（5%）
